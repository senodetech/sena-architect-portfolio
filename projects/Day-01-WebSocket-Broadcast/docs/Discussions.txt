	# Interview Answer (10/10)

	If tomorrow Microsoft asks

	> Why Express and WebSocket?

	I'd answer:

	> "Express handles traditional HTTP communication such as serving pages, REST APIs, routing, middleware, authentication, and request/response processing. However, HTTP is stateless and closes the connection after each response. A collaborative editor requires a persistent bi-directional connection where both the client and server can send messages at any time. WebSocket provides that persistent channel. Therefore Express manages the HTTP layer, while WebSocket manages the real-time communication layer."

	🔥

	That's a Senior Engineer answer.
